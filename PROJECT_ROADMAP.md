# Project Roadmap - Mechanical Restaurant Horror

## Timeline Overview
| Phase | Duration | Focus |
|-------|----------|-------|
| 1 - Foundation | Week 1-2 | Setup, Lobby, Day/Night |
| 2 - Core Gameplay | Week 3-4 | Food, Customers, Kill Mechanic |
| 3 - Night Mechanics | Week 5-6 | Animals, Windows, Challenges |
| 4 - Progression | Week 7-8 | 64 Days, Gems, Upgrades |
| 5 - Polish | Week 9+ | Sound, Visuals, Testing |

---

## Detailed Roadmap

### Week 1: Project Setup
- [ ] Initialize Roblox Studio project
- [ ] Set up version control (Git)
- [ ] Create basic place file structure
- [ ] Test Roblox Studio ↔ Git workflow

### Week 2: Lobby System
- [ ] Build lobby area (spawn point, UI)
- [ ] Implement player counter (1-10 players)
- [ ] Add "Friends Only" toggle
- [ ] Create teleport button to game
- [ ] Basic day/night cycle timer

### Week 3: Environment & Customers
- [ ] Design restaurant interior
- [ ] Create customer NPC model
- [ ] Implement customer spawning
- [ ] Basic ordering system (random food requests)

### Week 4: Food & Kill System
- [ ] Food cooking system (simple click-to-cook)
- [ ] Serving mechanic (give food to customer)
- [ ] "Weird customer" detection logic
- [ ] Kill button UI & function
- [ ] Wrong kill penalty system

### Week 5: Night Phase
- [ ] Transition day → night
- [ ] Mechanical animal models
- [ ] Animal spawning at night
- [ ] Basic AI (move toward players)

### Week 6: Defense Mechanics
- [ ] Window objects (lockable)
- [ ] Window locking UI/controls
- [ ] Challenge: outdoor food disposal
- [ ] Jumpscare effect & death

### Week 7: Progression
- [ ] Day counter (1-64)
- [ ] Gem collection system
- [ ] Upgrade shop UI
- [ ] Speed upgrade implementation
- [ ] Weapon upgrade implementation

### Week 8: Game Loop
- [ ] Win condition (survive 64 days)
- [ ] Lose condition handling
- [ ] Restart functionality
- [ ] Score/stats display

### Week 9+: Polish
- [ ] Sound effects (ambience, SFX)
- [ ] Background music
- [ ] Lighting improvements
- [ ] UI polish
- [ ] Bug fixing
- [ ] Playtesting

- Complex AI for mechanical animals
- Syncing game state across multiple players
- Performance with many NPCs
- Asset creation time

---

## Quick Wins (Do Early)
1. Placeholder cubes for all NPCs/objects
2. Simple text labels for UI
3. Basic movement scripts before polish
